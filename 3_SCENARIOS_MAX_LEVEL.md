# KỊCH BẢN 1: BÀN GIAO CODE 100% (ONE-TIME BUYOUT)

## Binh Pháp: 善戰者，無智名，無勇功 (The expert warrior wins without glory)
**Ý nghĩa**: Chiến thắng không ồn ào. Bán code = Exit nhanh, không rủi ro.

---

### A. ĐỊNH GIÁ CODE/IP (VALUATION)

#### Phương pháp 1: Cost-Based (Chi Phí Đầu Tư)
```
Thời gian phát triển: 3 tuần × 60h/tuần = 180 giờ
Rate công nghệ cao cấp: $100-150/giờ (thị trường US)
Cost basis: 180h × $125/h = $22,500

+ Giá trị IP (hệ thống AI, payment, retention): +$10,000
+ Risk premium (cofounder không biết code): +$5,000

TOTAL: $37,500 (Reasonable floor)
```

#### Phương pháp 2: Revenue Multiple (Dựa Trên Doanh Thu)
```
MRR hiện tại (Month 6 projection): $45,000
SaaS valuation: 3-5x ARR (early stage)

ARR = $45K × 12 = $540K
Valuation @ 3x ARR = $1.62M

Tech stack value = 40% of valuation (rule of thumb)
Code buyout = $1.62M × 40% = $648,000

Negotiated (50% discount vì chưa proven): $324,000
```

#### Phương pháp 3: Feature Replacement Cost
```
Nếu cofounder hire team để rebuild:
- Senior Full-Stack Dev: $150K/year
- AI/ML Engineer: $180K/year
- DevOps: $120K/year
- Timeline: 6 months (50% FTE)

Cost to rebuild: ($450K/year × 0.5 FTE × 0.5 year) = $112,500

+ Opportunity cost (6 months delay): $270K (45K MRR × 6)
TOTAL REPLACEMENT COST: $382,500
```

```
- 100 users cũ × $67 = $6,700 → Anh: $670
```

### B. HOÀN PHÍ SÀN (EXCHANGE REBATE BONUS)

**Context**: Nếu sàn (Binance, OKX) trả rebate về cho platform (thường 20-40% commission từ trading fees).

#### Rebate Revenue Model
```
User A trades $100,000 on Binance
Binance fee: 0.1% = $100
Rebate to Apex Rebate: 30% of $100 = $30

Distribution:
- Platform giữ: 50% = $15
- Anh nhận: 25% = $7.5
- User A nhận (cashback): 25% = $7.5
```

**Scaling Impact**:
```
Month 6: 1,000 users × $50K avg monthly volume = $50M
Trading fees @ 0.1%: $50,000
Rebate @ 30%: $15,000
Anh's share @ 25%: $3,750/month extra
```

**Total Month 6 Income (Scenario 2)**:
```
User % revenue: $5,192
Rebate share: $3,750
TOTAL: $8,942/month ($107K/year)
```

### C. CÔNG THỨC TỰ ĐỘNG

**File**: `src/lib/revenue-share-v2.ts`

```typescript
export function calculateUserBasedShare(month: string) {
  // 1. Classify users
  const usersOld = getUsersBeforeHandover(); // 100 users
  const usersNew = getUsersAfterHandover();  // 900 users
  
  // 2. Calculate tier
  const totalUsers = usersOld.length + usersNew.length;
  let newUserRate = 0.05; // Default 5%
  
  if (totalUsers > 20000) newUserRate = 0.12;
  else if (totalUsers > 5000) newUserRate = 0.10;
  else if (totalUsers > 1000) newUserRate = 0.075;
  
  // 3. Calculate revenue
  const oldUserRevenue = usersOld.reduce((sum, u) => sum + u.monthlyRevenue, 0);
  const newUserRevenue = usersNew.reduce((sum, u) => sum + u.monthlyRevenue, 0);
  
  const techAdvisorShare = (oldUserRevenue * 0.10) + (newUserRevenue * newUserRate);
  
  // 4. Add rebate share
  const rebateRevenue = getExchangeRebates(month);
  const rebateShare = rebateRevenue * 0.25;
  
  return {
    userShare: techAdvisorShare,
    rebateShare: rebateShare,
    total: techAdvisorShare + rebateShare,
  };
}
```

# KỊCH BẢN 3: PHƯƠNG ÁN TỐI ƯU MAX LEVEL (EQUITY + TOKEN + MULTI-STAGE)

---

### A. CẤU TRÚC ĐA TẦNG (MULTI-LAYER STRUCTURE)

#### Layer 1: Founding Equity (20-30%)
```
Anh: 25% equity in company
Cofounder: 60% equity
Reserved for employees/advisors: 15%

Vesting schedule:
- 4 năm vesting, 1 năm cliff
- Nếu anh rời trước 1 năm → 0% equity
- Sau 1 năm → 25% × (months worked / 48)
```

**Lý do**: 
- Standard Silicon Valley model
- Protects both parties (cofounder không lo anh rời sớm)
- Fair for long-term contribution

---

#### Layer 2: Revenue Share (5-10% Net Revenue)
```
Cơ chế:
- Anh nhận 7.5% Net Revenue monthly (thấp hơn 15% ở Option A)
- ĐỔI LẠI: Nhận 25% equity + token allocation

Ví dụ Month 12:
Net Revenue: $50K
Anh's monthly share: $50K × 7.5% = $3,750
Annual: $45K (thấp hơn Option A nhưng có equity)
```

**Binh Pháp**: 捨小取大 (Sacrifice small gains for bigger wins) - Revenue share thấp hơn, nhưng đổi lại equity.

---

#### Layer 3: Token Allocation (Future-Proof)

**Token Economics Design** (chuẩn bị từ bây giờ):

```
Total Token Supply: 100,000,000 APEX
Distribution:
- Team & Advisors: 20M (20%) → Anh: 5M (5%)

- Community Rewards: 25M (25%) → Staking, referrals, trading rewards

Anh's allocation: 5M APEX
Vesting: 4 năm, linear unlock
At $1/token → $5M value
At $10/token → $50M value 🚀
```

**Use Cases for APEX Token**:
1. **Staking**: Lock APEX → Get discount on fees (5-20%)
2. **Governance**: Vote on feature development
3. **Rewards**: Profitable users earn APEX
4. **Payment**: Pay subscription with APEX (10% discount)

**Binh Pháp**: 因利而制權 (Control the situation by creating the right conditions) - Token tạo loyalty và network effect.

# KỊCH BẢN 4: KHOÁN CHI PHÍ ỐM HẾT (FLAT OPERATING FEE)

## Binh Pháp: 簡則易從 (Simplicity Enables Execution)
**Ý nghĩa**: Đơn giản thì dễ thực hiện. Loại bỏ friction báo cáo → Focus vào growth.

---

### A. MÔ HÌNH KHOÁN CỐ ĐỊNH

**Nguyên tắc**: Anh ôm toàn bộ chi phí vận hành, cofounder trả 1 khoản cố định hàng tháng (không cần báo cáo từng xu).

#### Option 4A: Conservative Flat Fee
```
Fixed Monthly Fee: $800/month

Bao gồm:
- Vercel: ~$50
- Supabase: $25
- OpenRouter: $50-500 (anh chịu rủi ro nếu usage spike)
- Resend: $20
- Cloudflare: $10
- Sentry: $26
- Buffer/Contingency: $200

Anh's margin: $800 - $600 (avg cost) = $200/month profit

Cofounder lợi:
- Không cần track hóa đơn
- Fixed cost (predictable budget)
- Không lo anh "phình" chi phí
```

#### Option 4B: Aggressive Flat Fee (Higher margin)
```
Fixed Monthly Fee: $1,500/month

Lý do tăng:
- Anh chịu risk nếu AI usage explode ($500 → $2,000)
- Include "on-call support" (anh fix bugs 24/7)
- Premium for reliability guarantee (99.9% uptime SLA)

Anh's margin: $1,500 - $600 = $900/month profit
```

---

### B. KHOÁN LŨY TIẾN THEO SCALE

**Problem**: Fixed $800 không fair nếu có 10,000 users (chi phí lên $3,000).

**Solution**: Tiered Flat Fee

```
TIER 1 (0-500 users):     $800/month
TIER 2 (501-2,000 users): $1,500/month
TIER 3 (2,001-10,000):    $3,000/month
TIER 4 (10,001-50,000):   $6,000/month
TIER 5 (50,001+):         Negotiate (likely $10K-15K/month)
```

**Ví dụ**:
```
Month 1: 30 users → $800
Month 6: 1,000 users → $1,500
Month 12: 5,000 users → $3,000
Month 24: 20,000 users → $6,000
```

**Anh's margin tại scale**:
```
Month 24:
- Actual cost: ~$2,500 (Vercel Pro $100, Supabase Pro $500, OpenRouter $1,500)
- Flat fee: $6,000
- Margin: $3,500/month ($42K/year) ✅
```

---

### C. PROS & CONS

**Ưu điểm**:
- ✅ Cực kỳ đơn giản (1 số duy nhất mỗi tháng)
- ✅ Anh không cần gửi hóa đơn, screenshot
- ✅ Cofounder fixed cost, dễ budgeting
- ✅ Anh có margin profit (motivation to optimize costs)
- ✅ No disputes về "chi phí này có hợp lý không?"

**Nhược điểm**:
- ❌ Anh chịu risk nếu cost spike đột ngột (viral growth)
- ❌ Cần renegotiate tiers khi scale (có thể tranh cãi)
- ❌ Không có upside nếu company thành công lớn (chỉ có flat fee)

---

### D. COFOUNDER CÓ ĐỒNG Ý KHÔNG?

**Khả năng chấp nhận: 85%** ✅

**Lý do cofounder sẽ thích**:
1. **Predictable**: Budget cố định, dễ forecast
2. **Simple**: Không cần review hóa đơn hàng tháng
3. **Fair**: Tiers scale với users = cost scale với revenue
4. **Trust**: Anh có động lực optimize (margin của anh tăng khi cost giảm)

**Lý do có thể từ chối**:
1. **Expensive**: $800/month vs itemized $600/month = overpay $200
2. **Opacity**: Cofounder không biết anh kiếm bao nhiêu margin (nhưng this is the point!)

**Cách thuyết phục**:
> "Pay $800 flat, no hassle. Or pay itemized $600 + spend 5 hours/month reviewing invoices. What's 5 hours of your time worth? Probably $500+. So $800 is actually cheaper."

### C. TIERED PER-USER MODEL (Giảm theo scale)

**Problem**: $5/user × 100K users = $500K/month (quá cao, cofounder sẽ từ chối).

**Solution**: Giảm per-user fee theo scale (giống AWS volume discounts).

```
TIER STRUCTURE:
Users 1-1,000:       $5/user
Users 1,001-10,000:  $3/user
Users 10,001-50,000: $1.5/user
Users 50,001+:       $0.75/user
```

**Example at 100,000 users**:
```
First 1,000:   1,000 × $5 = $5,000
Next 9,000:    9,000 × $3 = $27,000
Next 40,000:   40,000 × $1.5 = $60,000
Last 50,000:   50,000 × $0.75 = $37,500
TOTAL: $129,500/month ($1.55M/year)

vs Flat $5/user: $500K/month (unrealistic)
```

**Cofounder's margin**:
```
Revenue: 100K × $67 = $6.7M/month
Anh's fee: $129.5K (1.93% of revenue)
Their net: $6.57M/month (still huge!)
```

### D. PROS & CONS

**Ưu điểm**:
- ✅ Tự động scale (no renegotiation needed)
- ✅ Fair alignment (anh's income grows with their success)
- ✅ Simple calculation (count users × $Y)
- ✅ Unlimited upside for anh
- ✅ Cofounder vẫn profitable (1-5% of revenue)

**Nhược điểm**:
- ❌ Có thể cao nếu users grow nhanh
- ❌ Anh chịu risk nếu churn cao (users giảm → income giảm)
- ❌ Cần define "active user" (paid? hoặc all signups?)

# KỊCH BẢN 6: MÔ HÌNH LAI ĐƠN GIẢN (SIMPLIFIED HYBRID)

## Binh Pháp: 兼收並蓄 (Embrace All, Own All)
**Ý nghĩa**: Kết hợp ưu điểm, loại bỏ nhược điểm.

---

### A. CẤU TRÚC LAI ĐỀ XUẤT

**Kết hợp Kịch bản 4 (Flat Fee) + Kịch bản 5 (Per-User)**:

```
BASE FEE: $500/month (covers infrastructure up to 500 users)

+ 

PER-USER FEE: $3/user for users beyond 500

Formula:
If users ≤ 500: Fee = $500
If users > 500: Fee = $500 + ((users - 500) × $3)
```

**Examples**:
```
Month 1: 30 users
Fee = $500 (base only)

Month 6: 1,000 users
Fee = $500 + ((1,000 - 500) × $3) = $500 + $1,500 = $2,000

Month 12: 5,000 users
Fee = $500 + ((5,000 - 500) × $3) = $500 + $13,500 = $14,000

Month 24: 20,000 users
Fee = $500 + ((20,000 - 500) × $3) = $500 + $58,500 = $59,000
```

---

### B. TẠI SAO LAI LÀ TỐI ƯU NHẤT?

**So với Flat Fee thuần**:
- ✅ Không overpay ở giai đoạn đầu ($500 vs $800)
- ✅ Scale fair hơn ($59K @ 20K users vs $6K flat fee)

**So với Per-User thuần**:
- ✅ Minimum guaranteed income ($500 base vs $60 @ 30 users)
- ✅ Simpler than tiered model

**So với % Revenue**:
- ✅ No need to track revenue (cofounder's secret sauce)
- ✅ Simpler calculation (count users)
- ✅ No disputes about "what is Net Revenue"
